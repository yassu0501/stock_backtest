# 日本個別株バックテストツール MVP仕様書

## 1. プロジェクト概要

| 項目 | 内容 |
|---|---|
| 目的 | 日本個別株の短期投資戦略（1週間〜1ヶ月）をブラウザでバックテスト |
| 技術スタック | HTML / CSS / Vanilla JS（フロント） + Node.js（プロキシサーバー） |
| データソース | Yahoo Finance（ticker: 7203.T 形式） |
| MVP戦略 | 移動平均クロス（拡張: RSI・ボリンジャーバンド） |

---

## 2. システム構成

```
[ブラウザ]
  └── index.html / style.css / app.js
        │ fetch("http://localhost:3000/api/stock?ticker=7203.T")
        ▼
[Node.js プロキシサーバー] server.js (port 3000)
        │ fetch Yahoo Finance CSV/API
        ▼
[Yahoo Finance]
```

**なぜプロキシが必要か**: ブラウザから直接Yahoo Financeを叩くとCORSエラー。Node.jsがCORS回避の中継役を担う。

## 5. APIエンドポイント設計（server.js）

### GET `/api/stock`

| パラメータ | 型 | 例 | 説明 |
|---|---|---|---|
| ticker | string | `7203.T` | Yahoo Finance形式の銘柄コード |
| period | string | `1y` / `2y` / `6mo` | 取得期間 |
| interval | string | `1d` | 日足固定（MVP） |

**レスポンス例:**
```json
{
  "ticker": "7203.T",
  "data": [
    { "date": "2024-01-05", "open": 2500, "high": 2550, "low": 2480, "close": 2530, "volume": 1200000 },
    ...
  ]
}
```

## 6. バックテストロジック（app.js）

### 移動平均クロス戦略

```
買いシグナル: 短期MA が 長期MAを 下から上に突き抜けた日（ゴールデンクロス）
売りシグナル: 短期MA が 長期MAを 上から下に突き抜けた日（デッドクロス）
ポジション:  常に1ポジション（ドテン売買）
手数料:      0.1%（買い・売り各）
```

### 計算する指標

| 指標 | 計算方法 |
|---|---|
| 総損益（円・%） | 最終資産 - 初期資金 |
| 勝率 | 利益トレード数 / 総トレード数 |
| 最大ドローダウン | 累積損益の山から谷への最大落差 |
| 平均保有日数 | 買い〜売りの日数平均 |
| 取引回数 | シグナル発生回数 |
| シャープレシオ | （任意・拡張） |

## 8. 注意事項・リスク

| リスク | 対策 |
|---|---|
| Yahoo Finance APIの仕様変更 | エラー時にCSVアップロードへフォールバック |
| 将来データの混入（look-ahead bias） | MAは当日終値まで使用、翌営業日に執行する設計 |
| 過剰最適化 | パラメータは固定プリセットのみ（任意値入力は拡張で） |
